# 🏆 家計簿AIレビューシステム プロジェクト完了報告書

本プロジェクトにおける全成果物の整理と、システム全体の品質評価を報告いたします。

## 1. 📂 成果物の整理

### 🏗️ コア・アーキテクチャ
- **統合エントリポイント (`main.py`)**: MF/Zaimのデータソース切り替え、スケジュール自動判定、マルチ出力を一元管理。
- **データモデル (`src/models.py`)**: Pydantic v2による厳格な型定義（Transaction, Asset, AIResponse）。
- **データベース層 (`src/db/database.py`)**: SQLiteによる永続化。重複防止（UPSERT）と分析用差分抽出ロジック。

### 📡 データ取得・分析部
- **Fetcher (`src/fetcher/`)**: ヘッダー解析を導入した高精度スクレイピング（MF）およびAPI連携（Zaim）。
- **AI分析 (`src/analyzer/`)**: Gemini 3.1 Pro Preview を活用。JSONモードによる構造化出力と、過去のサマリーを踏まえた継続的レビュー。
- **外部プロンプト (`prompts/`)**: システム/期間別の指示をMarkdownで外部化し、柔軟な調整が可能。

### 📢 出力・通知部
- **Slack通知 (`src/output/slack_notifier.py`)**: Block Kitを使用した視覚的にリッチなレポート送信。
- **Obsidian保存 (`src/output/obsidian_writer.py`)**: 年月フォルダ管理と、Index（目次）ファイルの自動更新機能。

### 🛠️ インフラ・自動化
- **環境構築スクリプト**: Windows/Linux/Raspberry Pi (32/64bit) に対応した `setup.sh`, `setup_raspberry_pi.sh` 等。
- **CI/CD (`.github/workflows/`)**: GitHub Actionsによる自動テスト(CI)と、ラズパイへの自動デプロイ(CD)。
- **デバッグツール (`tools/`, `inspect_db.py`)**: Slack単体テストやDB確認用ツール。
