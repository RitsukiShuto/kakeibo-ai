# 📋 Project Backlog: Kakeibo AI Review System

このプロジェクトの今後の改善案、新機能、および長期的なロードマップを管理するバックログです。

## 🚀 短期的な改善 (Quick Wins)
- [ ] **予算超過時のアラート通知**: Slack通知時に、予算を超過しているカテゴリを太字や絵文字で強調する。
- [ ] **カテゴリマッピングの自動化**: `settings.json` などで、特定の店名を特定のカテゴリに自動で振り分けるルール設定機能。
- [ ] **キャッシュ機能の導入**: 資産状況など、頻繁に変わらないデータの取得を最適化し、API/スクレイピングの負荷を軽減する。

## ✨ 新機能案 (New Features)
- [ ] **Slackからのオンデマンド分析実行**: SlackのSlash Command（例: `/review`）や「分析実行」ボタンを介して、任意のタイミングで家計簿の取得・分析をキックできる機能。
- [ ] **週次/月次の予実比較グラフ**: Visualizerを拡張し、資産推移だけでなく、予算に対する支出の推移をバーチャートで表示する。
- [ ] **マルチユーザー対応**: 同居家族などで、複数のプロファイルを切り替えて分析できる機能。
- [ ] **レシートOCR連携**: 紙のレシートを写真に撮って、特定のフォルダに置くだけで自動でDBに取り込む機能。
- [ ] **「ととのい指数」の長期統計**: 過去のスコアを蓄積し、自分の家計管理がどれくらい安定してきたかを可視化する。

## 🧠 AI分析の高度化
- [ ] **特定カテゴリの深掘り分析**: 特定の月だけ「食費」が異常に高い場合、AIがその内訳（外食が多いのか、特定のスーパーでの買い出しが多いのか）を自動で深掘りする。
- [ ] **ライフプランシミュレーション連携**: 現在の貯蓄ペースに基づき、「N年後のFIRE達成確率」や「住宅購入の妥当性」などをAIがアドバイスする。
- [ ] **ギャル以外のキャラクター追加**: ユーザーの好みに合わせて、執事、軍曹、癒やし系など、AIのキャラクター（システムプロンプト）を選択可能にする。

## 🛠 システム・インフラ
- [x] **Webダッシュボード**: Obsidianだけでなく、ブラウザでいつでも家計状況を確認できるシンプルなWeb UI (Streamlit) の構築。
- [ ] **コンテナ化 (Docker)**: Raspberry Pi等へのデプロイをより簡単にするための Dockerfile および docker-compose の整備。
- [x] **テストカバレッジの向上**: 分析ロジックやデータ取得部分のユニットテストを拡充し、信頼性を高める。

## 📅 ロードマップ案
- **Phase 1 (現在のフェーズ)**: データの自動取得とAIによる基本的なレビュー・予実管理。
- **Phase 2**: 予実の視覚化強化と、AIによるアクション提案の精度向上。
- **Phase 3**: ユーザー体験の向上（WebUI、複数ソース連携の簡略化）。
- **Phase 4**: ライフプランニングと統合された高度なパーソナル・ファイナンス・エージェント化。
