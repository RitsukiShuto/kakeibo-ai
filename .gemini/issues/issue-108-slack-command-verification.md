# Issue: Slackコマンドの検証と実機連携の確認 (Issue #108)

## 概要
現在実装されている Slack notifier および Slack server (`src/output/slack_server.py`, `slack_notifier.py`) が正しく動作するかを検証し、実機（スマートフォン/PC）からの Slack コマンド実行に対するレスポンスが正常に返ることを確認します。

## 背景
- `src/output/slack_server.py` による Slack コマンド受付機能は実装済みだが、エンドツーエンドでの動作検証が不十分。
- ストリーミングレスポンスや、分析結果の通知が適切なフォーマットで表示されるかを確認する必要がある。
- ステージング環境での検証を行い、本番デプロイに向けた確信を得る。

## 完了定義 (Definition of Done)
- [ ] ローカルおよびステージング環境において、Slack App からのコマンド（`/kakeibo` 等）に対してバックエンドが正常に応答する。
- [ ] 分析結果が Slack 上で正しくフォーマット（リッチテキスト、ブロック、または整形されたテキスト）されて表示される。
- [ ] エラー時のハンドリングが適切に行われ、ユーザーにわかりやすい通知が飛ぶ。
- [ ] `qa-tester` による通信テストをパスする。
