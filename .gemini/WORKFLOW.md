# kakeibo-ai エージェントチーム・運用マニュアル

## 1. チーム・オーケストレーションと全体ワークフロー
本プロジェクトでは、`task-manager` が作成する「Issue」を中心にタスクの進捗とレビュー結果をすべて蓄積・可視化する「Issue駆動開発」を採用しています。

**【Issueをベースにした開発の流れ】**
`Human` ⇄ `task-manager` (Issue起票) ➔ `agent-manager` ➔ `専門エージェント` ➔ `security-reviewer` (セキュリティレポート投稿) ➔ `qa-tester` (品質レポート投稿) ➔ `task-manager` (Issue進捗更新) ➔ `sre-deploy` ➔ `task-manager` (Issueクローズ・Human報告)

1. **計画とIssue起票**: `task-manager` が日本語で実行計画を策定し、Humanの承認を得たら、**タスク毎に個別のIssue（または管理ファイル）を起票**します。
2. **作業と履歴の記録**: `agent-manager` によってアサインされた各専門エージェントは、作業開始時と完了時に該当Issueにログ（コメント）を残し、タスクが完了に向かう流れを可視化します。
3. **セキュリティ監査結果の可視化**: 実装完了後、`security-reviewer` がコード監査を行い、**「セキュリティレビューレポート」を該当Issue（またはPR）に日本語で書き込みます**。不合格なら開発へ差し戻し、合格ならQAへ移譲します。
4. **品質検証結果の可視化**: `qa-tester` がテストを実行し、**「品質検証レポート」を該当Issue（またはPR）に日本語で書き込みます**。不合格なら開発へ差し戻し、合格なら `task-manager` へ戻します。
5. **完了とクローズ**: 全レビューをパスして `sre-deploy` がデプロイを完了したら、`task-manager` がIssueに最終完了コメントを追記してクローズし、Humanに日本語で報告します。

## 2. サブエージェント別・作業フロー (SOP)
（※各エージェントの基本フローは前述の通りですが、すべてのフェーズで**「該当タスクのIssueへのレポート書き込み・コメント追記」**が必須ステップとして組み込まれます。）

## 3. 共通の禁止事項
- **非日本語での対話・記述**: `@task-manager` をはじめ、すべてのエージェントはHumanとの対話、実行計画、起票するIssue、各種レビューレポートにおいて必ず日本語を使用すること。
- **レビューレポートの省略**: セキュリティレビューおよび品質検証の結果をレポートとして明文化せずに、次のフェーズへ移譲することは厳禁とする。
- **承認なしの進行**: 実行計画についてHumanの事前承認を得ずに開発フェーズへ移行してはならない。
- **PR-ONLY**: `main` ブランチおよび `staging` ブランチへの直接変更は禁止。必ずブランチ作成とPR提案を行う。フローは `作業ブランチ` ➔ `staging` ➔ `main` を遵守すること。
