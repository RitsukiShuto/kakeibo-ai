ダッシュボードの全面刷新および致命的バグの改修（Phase 1・Phase 2）は完了しましたが、その後の全画面監査および事前のレビューにて、以下のUX改善項目や技術的負債が未解決のまま残っています。今後の保守性・操作性向上のため、本Issueにてトラッキングします。

## 🎯 1. UI/UX 改善（Phase 3）

- [ ] **AIモデル選択のカスタムダイアログ化 (High)**
  - 現在 `window.confirm` を使用しているため、モバイル等でポップアップブロックの対象になり得る。Reactの状態を用いたカスタムモーダルに置き換える。
- [ ] **モバイル端末でのサイドバーレイアウト最適化 (Medium)**
  - 画面幅が狭い場合、左側のサイドバーが上部に詰まりメインコンテンツを圧迫する。ボトムタブメニュー、もしくはハンバーガーメニューに変更する。
- [ ] **AIレポートのObsidian記法レンダリング対応 (Low)**
  - AIが生成する `> [!TIP]` （コールアウト記法）や `![[image]]` 記法がプレーンテキストとして表示されている。`react-markdown` のカスタムレンダラーを追加して装飾する。
- [ ] **「トップへ戻る」ボタンの追加 (Low)**
  - 明細一覧画面 (`Transactions.tsx`) などで無限スクロールを繰り返した際、ページ上部に簡単に戻れるフローティングボタンを追加する。
- [ ] **テーブルのソート機能実装 (Low)**
  - 明細一覧テーブルのヘッダー（日付、金額、大項目など）をクリックして昇順/降順を切り替えられるようにする。
- [ ] **サイドバー下部の「Slack Bot」ステータス表示位置の修正 (Low)**
  - 画面コンテンツが長いページでサイドバーの下部が見切れてしまうため、`sticky` もしくは `fixed` 配置で常に表示されるようにする。
- [ ] **ユーザーアイコンのメニュー実装 (Low)**
  - 右上の人型アイコン（`TopHeader`）が現在ダミーになっているため、プロフィール設定への遷移やログアウトのドロップダウンメニューを追加する。

## 🛠 2. アーキテクチャ・コード品質の改善

- [ ] **`any` 型の排除とTypeScript型の厳密化 (High)**
  - `Dashboard.tsx`, `Settings.tsx`, `ExpenseSplitter.tsx` で `useState<any>` が残存している。APIからのレスポンス型（`Budget`, `Profile` 等）を正確に定義し、型安全性を高める。
- [ ] **設定API (`PUT /api/settings/*`) のバリデーション強化 (Medium)**
  - 任意のJSONを受け入れてファイルに書き込める状態になっているため、Pydanticモデルを用いた入力検証を導入する。
- [ ] **CORSの制限 (Medium)**
  - バックエンドAPIのCORS設定が `allow_origins=["*"]` となっているため、デプロイ環境に合わせて適切なオリジンに制限する。
- [ ] **Rechartsのコンソール警告解消 (Low)**
  - 初期レンダリング時に `The width or height of the container is -1` という警告が出るため、親要素の最小サイズ指定やマウントタイミングの調整を行う。
- [ ] **不要なCSSのクリーンアップ (Low)**
  - `App.css` などにViteデフォルトの未使用スタイル（`.hero`, `.counter` など）が残存しているため削除する。

## 📝 備考
これらのタスクはシステム自体の動作を止めるものではありません。優先度順（High → Low）に、または関連するコンポーネントを修正するタイミングで順次消化していくことを推奨します。
