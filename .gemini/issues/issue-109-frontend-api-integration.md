# Issue: フロントエンド統合とデータ追加・削除機能の実装 (Issue #109)

## 概要
Webダッシュボード（フロントエンド）をバックエンドAPIと統合し、リアルタイムでのデータ表示を可能にします。また、ユーザーがUI上から直接取引データを追加・削除できる機能を実装し、家計簿としての基本操作を完結させます。

## 背景
- 現在のフロントエンドはプロトタイプであり、バックエンドの実データとの連携が不十分。
- データの自動取得だけでなく、手入力による補正や誤ったデータの削除など、CRUD操作の基本部分が必要。
- Phase 1 のマイルストーンとして、ダッシュボードの機能性を向上させる。

## 完了定義 (Definition of Done)
- [ ] フロントエンドがバックエンドAPIから家計簿データを取得し、グラフやリストとして正しく表示される。
- [ ] 取引データの新規追加フォームが動作し、DBに保存される。
- [ ] 取引データの削除機能が動作し、DBから削除（または論理削除）される。
- [ ] UI上の操作結果が即座にビューに反映される。
- [ ] `qa-tester` による統合テストをパスする。
