プロジェクトの現状を分析し、次に取り組むべきタスクを日本語で提案します。

以下の手順で分析してください：

1. `PROJECT_BACKLOG.md` を読み、ロードマップ上の未完了タスクを把握する
2. `git log --oneline -20` で直近の作業履歴を確認する
3. `gh issue list --state open` でオープンなIssueを確認する（使用可能な場合）
4. `src/` と `frontend/` の構成を俯瞰し、技術的負債を特定する

以下の優先軸で次タスク候補を3〜5件提案する：
- 他の作業をブロックしているもの
- Phase 2〜4のバックログ項目
- `PROJECT_BACKLOG.md` の「今後の技術改善タスク」

出力フォーマット（日本語）：

```
## 次タスク提案

### 1. [タスク名]
- **目的**: ...
- **対象**: `src/xxx.py` など
- **難易度**: 小 / 中 / 大
- **推奨優先度**: 高 / 中 / 低
- **理由**: ...

### 2. [タスク名]
...
```
