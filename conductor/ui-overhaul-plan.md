# UI/UX 全面刷新 デザイン仕様書 (Vibrant Minimalist Overhaul)

## 1. 目的
家計管理アプリの体験を「管理」から「対話とインサイト」へ昇華させるため、既存のカード型UIを廃止し、ドキュメントのように流れる「Vibrant Minimalist」デザインへと全面的に刷新する。あわせて、低価値な機能を削除し、情報の密度とアクションの直感性を高める。

## 2. デザインコンセプト: Vibrant Minimalist
- **ダークテーマ**: 深いネイビー（Slate-950）を基調とし、鮮やかなアクセントカラー（Indigo, Amber, Emerald）を使用。
- **大胆なタイポグラフィ**: 数字や見出しを大きく、力強く表示し、視覚的階層を明確にする。
- **枠線の排除**: 従来のカード（囲み）を極力減らし、余白とフォントウェイトでセクションを区切る。
- **高密度な情報配置**: 大画面を活かしたマルチカラムレイアウトと、情報の重要度に応じたサイズ調整。

## 3. 画面別の刷新仕様

### 3.1 Dashboard (ダッシュボード)
アプリの核となる画面。高レベルなサマリーから具体的なアクションまでを1画面に集約する。
- **Overview (最上部)**:
    - 合計支出、1日平均、家計スコアを巨大なフォントで横並び表示。
    - 同行の右端に「資産構成（ドーナツチャート）」を配置。
- **AI Insights**:
    - AIによる最新の分析結果を、プレーンテキストで2行程度表示。背景や枠線はなし。
- **Cash Flow Analysis (Sankey)**:
    - 収入源 → 総収入 → カテゴリ（固定/変動） → 詳細（家賃/食費等）の多階層フローを視覚化。
    - 要素が重なり合わないよう、高さを十分に確保（h-[450px]程度）。
- **Operations (中段)**:
    - **予算ペースメーカー**: 
        - 週次/月次/四半期/年次の4段階切り替えトグル。
        - 各カテゴリの進捗バーに「今日の理想ライン（オレンジ）」を表示。
        - 「Weekly Form（勝敗表示）」: W/L/D形式で直近の成績を表示。進行中の週を鮮やかな枠線で強調。
    - **直近の明細**: 最小限の情報を高密度にリスト表示。
- **Reimbursements (最下部)**:
    - 精算待ちリストとAI立替検知を横並びで配置。

### 3.2 Transactions (入出金明細)
- **デザイン**: 視認性を重視した大きな明細テーブル。
- **構成**:
    - 上部に「すべて/支出/収入」の切り替えと検索・フィルタ。
    - 金額は大きく、カテゴリは鮮やかなタグ形式。
    - ホバーエフェクトによる行の強調。

### 3.3 AI Review (AIレポート履歴)
- **デザイン**: 「歴史」を感じさせるクロノロジカル（時系列）なリスト。
- **構成**:
    - 最新のレポートを「特大カード」として最上部に配置。
    - 過去のレポートはスコアと要約のみのグリッド表示。
    - レポート詳細をクリックでシームレスに開く。

### 3.4 Settings (設定)
- **構成**: 
    - 不要なマネーフォワードアカウント設定等を削除。
    - **月間予算設定**: 各カテゴリの数値を入力するシンプルなフォーム。
    - **AIペルソナ設定**: ギャル/執事/禅などのキャラを「特大選択カード」で切り替え。
    - **システム設定**: 通知や同期のON/OFFをトグルで管理。

## 4. 機能の削除 (Deprecation)
以下の機能・ページは刷新後のスコープから除外し、関連コード（Frontend/Backend）をクリーンアップする。
- **AI Chat**: 自由対話機能。
- **Life Plan**: 長期資産シミュレーション。
- **Expense Splitter (独立ページ)**: ダッシュボードへ機能を統合するため、ページとしては削除。
- **MoneyForward Account Config**: UI経由でのアカウント設定機能。

## 5. バックエンド改修要件 (Backend Requirements)

### 5.1 新規・拡張エンドポイント
1. **`/api/stats/flow` (新規)**:
    - Sankey Diagram用の多階層データを生成。
    - `nodes`: [{id, name}], `links`: [{source, target, value}] 形式。
2. **`/api/budget-actual` (拡張)**:
    - クエリパラメータ `timeframe` (weekly, monthly, quarterly, yearly) に対応。
    - 各期間における予算、実績、およびPace Limit（今日時点の理想値）を計算して返す。
3. **`/api/analysis-history/latest-summary` (新規)**:
    - 最新のAIレポートから、プレーンテキストの要約（2行程度）のみを返す。
4. **`/api/analysis-history/form` (新規)**:
    - 予算達成状況に基づく勝敗データ（W, L, D, -）の配列を取得。

## 6. 技術スタック
- **Frontend**: React, TypeScript, Tailwind CSS, Lucide React, Recharts (Sankey/Area/Pie)
- **Backend**: FastAPI, SQLAlchemy (SQLite)
